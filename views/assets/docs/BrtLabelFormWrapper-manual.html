BrtLabelFormWrapper mpbrtlabels

Manuale tecnico (API JS) per la classe BrtLabelFormWrapper. Questo documento descrive: proprietà, metodi, prerequisiti DOM e flussi principali.

1) Scopo

BrtLabelFormWrapper è un wrapper JS che gestisce:

  • indicizzazione e lettura/scrittura dei campi del form (in particolare table-input e select)
  • caricamento iniziale dati dal server (readOrderRequestParameters)
  • toggle di visibilità per i campi COD (contrassegno)
  • sincronizzazione forzata di changeOrderState quando si è in Admin Orders
  • gestione colli (tabella parcels): aggiunta riga, volume, totali, salvataggio
  • invio richiesta a BRT e parsing risposta
  • merge e visualizzazione in browser dei PDF etichette (base64) tramite PDFLib
  • helper dialog (open/close) e container growl

2) Firma costruttore

new BrtLabelFormWrapper(endpoint, orderId, action, form, isAdminOrdersPage = false, autosave = false)
Parametro Tipo Descrizione
endpoint string URL del controller admin (AJAX). Esempio: index.php?controller=AdminMpBrtLabels&token=...
orderId string|number ID ordine corrente (usato in molte action)
action string Action AJAX di default per write/select autosave (es. saveLabelField)
form HTMLFormElement Elemento <form> contenitore (es. #form-brtlabel)
isAdminOrdersPage boolean|string Se truthy, forza changeOrderState su ON. Dal Twig spesso arriva come stringa ("1", "true") e viene normalizzata internamente.
autosave boolean Se true, i <select name> vengono salvati al change via AJAX

3.2 Campi gestiti tramite data-category="createData"

Il wrapper indicizza tutti gli elementi in form che hanno data-category="createData". Per determinare il nome usa, in ordine:

  • el.dataset.name
  • el.getAttribute("name")
  • el.name

Questi elementi vengono raccolti in: createDataElements e mappati in createDataByName: Map<string, HTMLElement[]>.

3.3 Web component <table-input>

Il wrapper gestisce principalmente table-input e legge/scrive preferendo rawValue quando presente. Per gli switch usa data-type="switch" + data-value-on/data-value-off.

3.4 Tabella colli

  • Tabella: #mpbrtlabel-table-parcels
  • Tbody: #mpbrtlabel-table-parcels tbody (il server può ritornare HTML da rimpiazzare)
  • Totali: #totColli, #totPeso, #totVolume

5) Metodi pubblici (uso tipico)

Metodo Firma Descrizione
refresh async refresh() Reindicizza il form, ribinda eventi, e ricarica parametri da server

6) Metodi interni / helper

Questi metodi sono pensati per uso interno (prefisso _), ma sapere cosa fanno aiuta a capire il comportamento.

  • _index() — indicizza elementi e rebind delle funzioni UI/COD/changeState
  • _bindUiButtons() — collega i click dei bottoni UI ai metodi createRequestParcel(), onBtnClose(), ecc.
  • _bindCodToggle(), _updateCodVisibility() — show/hide righe COD in base a isCODMandatory
  • _bindChangeOrderStateSync(), _syncChangeOrderState(), _isAdminOrders() — forza changeOrderState quando Admin Orders
  • _request(mode, extra, actionOverride) — wrapper fetch che invia ajax=1, action, mode e ritorna JSON
  • _getTableInputType(), _setTableInputValue(), _setAllSwitches() — gestione coerente dei table-input
  • _base64ToUint8Array() — decode base64
  • _mergePdfStreamsAndOpen(), _openPdfBytes() — merge PDF e apertura in nuova scheda

7) Esempi d’uso

7.1 Inizializzazione (Admin Order Page)

// tipico in Twig/JS dopo DOMContentLoaded
const form = document.getElementById('form-brtlabel');
window.BrtLabelForm = new BrtLabelFormWrapper(
  MPBRTLABELS_ENDPOINT,
  MPBRTLABELS_ORDER_ID,
  'saveLabelField',
  form,
  MPBRTLABELS_ADMINORDERS,
  false
);

7.2 Aprire/chiudere dialog

window.BrtLabelForm.showBrtLabelDialog();
// ...
window.BrtLabelForm.hideBrtLabelDialog();

7.3 Ottenere payload createData e inviare request

const payload = window.BrtLabelForm.readCreateData();
console.log(payload);

await window.BrtLabelForm.createRequestParcel(0); // invia e poi parseResponse

7.4 Visualizzare JSON richiesta (debug)

await window.BrtLabelForm.createRequestParcel(1);

8) Flusso PDF etichette

Quando la risposta include una struttura come (semplificata): 

{
  "executionMessage": { "code": 0, "codeDesc": "OK", "message": "..." },
  "labels": {
    "label": [
      { "stream": "JVBERi0xLjcKJc... (base64)" },
      { "stream": "JVBERi0xLjcKJc... (base64)" }
    ]
  }
}

allora parseResponse():

  • estrae tutti gli stream
  • li carica con PDFLib
  • li unisce
  • apre il PDF risultante in una nuova scheda

9) Note operative / limitazioni note

  • Popup blocker: la UI prova ad aprire il PDF in una nuova scheda. Se bloccato dal browser, il wrapper mostra un errore.
  • Struttura labels: attualmente il codice assume data.labels.label come array. Se l’API cambia e ritorna labels come array diretto, va adeguato.
  • Delete parcel: deleteParcel è stub (vuoto).
  • Print parcel: onBtnPrintParcel è stub (vuoto).
