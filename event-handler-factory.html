Factory che restituisce un event handler (pattern per gestire meglio gli eventi)

Quando usi addEventListener e removeEventListener, la regola pi importante : per rimuovere un listener devi passare la stessa identica funzione (stessa reference) che avevi aggiunto.

1) Il problema: arrow inline = impossibile fare remove

Questo non rimuove niente, perch la arrow  una funzione nuova: 

btn.addEventListener('click', () => {
  console.log('click');
});

btn.removeEventListener('click', () => {
  console.log('click');
}); // ❌ non funziona (reference diversa)

2) Soluzione base: funzione nominata (reference stabile)

function onClick(e) {
  e.preventDefault();
  console.log('click');
}

btn.addEventListener('click', onClick);
btn.removeEventListener('click', onClick); // ✅ funziona

3) Quando serve una "factory" che restituisce un handler

Spesso vuoi che l'handler abbia accesso a dati extra (es. una row, un id, un riferimento a un input, ecc.). Un modo pulito  usare una factory: una funzione che crea e restituisce l'handler. 

// factory: prende row e restituisce la callback reale
const onRowBlurFactory = (row) => (e) => {
  // e  l'evento (arriva dal browser)
  // row  "catturata" nella closure
  console.log('Blur su', e.target, 'nella row', row);
};

const handler = onRowBlurFactory(row);
input.addEventListener('blur', handler);
input.removeEventListener('blur', handler); // ✅ funziona

4) Perch in quel caso non puoi chiamare la factory ogni volta

Se fai input.addEventListener('blur', onRowBlurFactory(row)) ogni volta, stai creando una nuova funzione ogni volta. Quindi poi removeEventListener non matcha e non rimuove. 

input.addEventListener('blur', onRowBlurFactory(row));
input.removeEventListener('blur', onRowBlurFactory(row)); // ❌ NON rimuove

5) Pattern pratico: salvare l'handler sull'elemento (come nel tuo codice)

Per avere una reference stabile, puoi salvarla come propriet sull'elemento DOM. 

// crea una volta sola e riusa
btn._onAddParcel ??= onAddParcelFactory();

btn.removeEventListener('click', btn._onAddParcel);
btn.addEventListener('click', btn._onAddParcel);

function onAddParcelFactory() {
  return function (e) {
    e.preventDefault();
    // ... logica
  };
}

6) Cosa significa ??=

x ??= y significa: "assegna y a x solo se x  null o undefined". 

// equivalente
if (btn._onAddParcel === null || btn._onAddParcel === undefined) {
  btn._onAddParcel = onAddParcelFactory();
}

7) Come passa l'evento (event) in questo pattern?

Non lo passi tu: lo passa il browser quando l'evento accade. Quindi l'handler che restituisci deve avere la firma (e) => { ... }. 

const handler = (e) => {
  console.log(e.type); // 'click', 'blur', 'input', ...
};

btn.addEventListener('click', handler); // il browser chiamer handler(e)
